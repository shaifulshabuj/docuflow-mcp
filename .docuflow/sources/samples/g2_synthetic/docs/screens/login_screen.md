# ログイン画面 基本設計書

## 画面ID
SCR-AUTH-001

## 概要
ユーザーがシステムにログインするための画面。

## 画面項目レイアウト（Excel風）

| 項目名 | 種別 | 必須 | 備考 |
|---|---|---|---|
| ユーザー名 | テキストボックス | 〇 | 最大50文字 |
| パスワード | パスワードボックス | 〇 | |
| ログインボタン | ボタン | - | クリックで認証処理を実行 |
| パスワード忘れリンク | リンク | - | パスワード再発行画面へ遷移 |

## 処理フロー
1. ユーザー名とパスワードを入力してログインボタンを押下する。
2. 認証APIを呼び出す。
3. 成功した場合、ダッシュボード画面へ遷移する。
4. 失敗した場合、「認証に失敗しました」とエラーメッセージを表示する。
