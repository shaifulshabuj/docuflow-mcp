# DocuFlow v1.6.0 — Philosophy Reset

> *"Every tool built to manage work eventually becomes work itself."*
> Intent in → Value out → Nothing in between.

## Why this release exists

By v1.5.2, DocuFlow had drifted into the very anti-pattern its [LinkedIn post on the Tracker's Paradox](https://www.linkedin.com/posts/shaifulshabuj_softwareengineering-productdevelopment-softwarearchitecture-ugcPost-7461730048954474496-ASZF) warns about: a tool that exists to maintain itself. The repo's own dogfood wiki had grown to **414 entity pages**, most of them noise — single English words extracted from bullet lists, emoji-led section dividers captured as entities, file references masquerading as concepts. `wiki_search` would return `entity_any` and `entity_add` for half its hits. The "answers compound" promise was breaking.

v1.6.0 is the first release of a multi-version reset. It does not yet remove features — that's planned for v1.7 (soft deprecation) and v2.0 (package split). What it does is **restore the value-out promise** so the rest of the reset has a foundation worth keeping.

## Empirical proof on this repo

Ran `docuflow rewiki` on `.docuflow/` (the repo's own wiki). Measured:

| Metric | Before (v1.5.2) | After (v1.6.0) | Δ |
|---|---:|---:|---:|
| Entity pages | 256 | **96** | **−62.5%** |
| Concept pages | 17 | 14 | −18% |
| Synthesis pages | 15 | 13 | −13% |
| Total wiki pages | 283 | 123 | −57% |
| Unit tests | 27 | **76** | +49 |
| Single-word generic entities (`architecture`, `documentation`, `overview`, …) | 11 | **0** | — |
| `synthesiss/` typo directory | present | gone | — |

The remaining 96 entity pages are real signal: 15 MCP tools, 10 CLI commands, 4 packages, 11 version labels, 6 security concepts, plus genuine product/concept names (`llm_wiki_pattern`, `claude_desktop`, `copilot_cli`, `mcp_sdk`, etc.).

## Upgrade path

**Existing users:** no breaking changes. After upgrading:

```bash
npm install -g @doquflow/cli@1.6.0
cd /path/to/your/project
docuflow rewiki --dry-run    # see the predicted cleanup
docuflow rewiki              # backs up, re-ingests with tightened rules
```

The command preserves user-saved synthesis pages (those produced by `save_answer_as_page`), backs up the entire wiki to `.docuflow/wiki.backup-<timestamp>/` by default, and writes a full audit report to `.docuflow/rewiki-report.md` listing every removed page with its rejection reason.

If anything looks wrong, the backup is one `mv` away.

## What's next

v1.6.0 was the **quality pass**. The remaining milestones in the reset:

- **v1.7 — Soft deprecation.** Re-label `watch`, `sync --ai`, `ui`, `lint`, `recent`, `review` as Advanced. Core surface becomes `init`, `status`, `query`, `ingest`. Zero command removal.
- **v2.0 — Package split.** `@doquflow/core` (irreducible MCP server) and `@doquflow/studio` (UI/REST/daemons). `@doquflow/cli` stays as a meta-package — existing install path unchanged.
- **v3.0 — Evidence-based pruning.** If `studio` features show genuine use, they earned their keep. If not, they retire.

Tracked under [umbrella issue #1](https://github.com/shaifulshabuj/docuflow-mcp/issues/1).

## The "no" list until v2.0

Every PR has to answer one question in one sentence: *"This makes the user's problem easier, not our system easier to build."*

- ❌ No new MCP tools
- ❌ No new CLI subcommands
- ❌ No new UI views
- ❌ No new integrations
- ✅ Extractor precision, schema authoring UX, query-result quality, value-out docs

DocuFlow is returning to its core. Thanks for sticking with it through the drift.
