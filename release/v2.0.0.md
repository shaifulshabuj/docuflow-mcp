# DocuFlow v2.0.0 — Package Split

> *"Every tool built to manage work eventually becomes work itself."*
> Intent in → Value out → Nothing in between.

## Why this release exists

[v1.7.0](v1.7.0.md) was the **surface labelling pass** — 5 core commands front and centre, 9 operational commands under `advanced`, every old path preserved. It answered *where* each command belongs.

v2.0.0 is the **package split** — the monolithic `@doquflow/server` has been split into focused packages so a lean deployment can exist. A server-only install no longer pulls in the UI, REST API, and daemon machinery it doesn't need. The CLI and install path are unchanged.

## The bright line holds

**Zero breakage for existing users.** Every existing install command, CLI invocation, `.mcp.json` registration, and MCP host configuration continues to work without any user action.

```bash
# All still work exactly as before:
npm install -g @doquflow/cli
npx @doquflow/server
docuflow watch
docuflow sync --ai
```

## What landed

Five issues, five PRs, all squash-merged onto main:

| PR | Issue | What it does |
|----|-------|--------------|
| #34 | #28 | `@doquflow/core` — 4 core MCP tools extracted as their own publishable package |
| #35 | #29 | `@doquflow/studio` — 11 advanced tools + UI + REST API; canonical MCP binary at `dist/mcp/index.js` |
| #36 | #30 | `@doquflow/server` — thin back-compat shim that re-exports studio; zero breakage |
| #37 | #31 | `@doquflow/cli` — converted to meta-package; depends on core + studio directly, drops server dep |
| #38 | #32 | `docuflow doctor` — new diagnostic command for install, MCP registrations, and wiki health |

## The new package layout

```
@doquflow/core      4 MCP tools (query_wiki, ingest_source, wiki_search, read_module)
@doquflow/studio    11 advanced tools + UI + REST API + MCP binary
@doquflow/server    Back-compat shim → re-exports @doquflow/studio
@doquflow/cli       Meta-package CLI (depends on core + studio)
```

All 15 MCP tools remain available via any install path.

## Upgrade path

```bash
npm install -g @doquflow/cli@2.0.0
```

No commands change paths. Optionally re-run `docuflow init` to refresh your MCP registration to the new canonical studio binary path:

```bash
cd /path/to/your/project
docuflow init   # safe to re-run — updates only the DocuFlow MCP registration
```

Run `docuflow doctor` to confirm everything is wired correctly.

## Install options in v2.0

```bash
# Full CLI (recommended — installs core + studio + all 15 MCP tools):
npm install -g @doquflow/cli

# Core only (4 MCP tools, smallest install, no UI/REST/daemon):
npm install -g @doquflow/core

# Studio (11 advanced tools + UI + REST API + MCP binary):
npm install -g @doquflow/studio

# Back-compat (same as studio, for existing .mcp.json registrations):
npx @doquflow/server     # still works
```
