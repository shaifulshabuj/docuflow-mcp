# DocuFlow v1.7.0 — Soft Deprecation

> *"Every tool built to manage work eventually becomes work itself."*
> Intent in → Value out → Nothing in between.

## Why this release exists

[v1.6.0](v1.6.0.md) was the **quality pass** — extractor tightening, wiki cleanup, 256 → 96 entity pages on the dogfood. It restored the *value-out promise* of the wiki without changing the user surface.

v1.7.0 is the **surface labelling pass**. The CLI had grown to 11 subcommands, with the irreducible core (`init` / `status` / new `query` / new `ingest` / `rewiki`) buried under operational scaffolding (`watch`, `sync --ai`, `ui`, `review`, `recent`, `suggest`, `update`). New users met the full surface before they met the value pipe.

The fix isn't deletion — that breaks existing users and v1.6.0's philosophy reset is a **multi-version arc, not a rug pull**. The fix is *relabelling*: surface the core, group the rest under `advanced`, and keep every old path working.

## The bright line

**Zero command removal through v2.x.** Every existing `docuflow watch`, `docuflow sync --ai`, `docuflow ui`, `docuflow review`, etc. keeps working at its old top-level path. The new `advanced` prefix is *optional* everywhere — `docuflow advanced watch` and `docuflow watch` invoke the same code.

Scripts, CI jobs, git hooks, IDE integrations: none of them need to change.

## The new surface

```
$ docuflow --help

DocuFlow — preserve decision context for AI agents.

CORE
  docuflow init                         Initialise .docuflow/ in this project
  docuflow ingest <file>                Add a source document to the wiki
  docuflow query "<question>"           Ask the wiki — answer with citations
  docuflow status                       Show wiki health and counts
  docuflow rewiki                       Migrate / re-ingest with current rules

ADVANCED
  docuflow advanced --help              See watch / sync / ui / review / recent / suggest / update

  docuflow --version                    Print version
  docuflow --help                       Show this help
```

The advanced surface (one layer deeper) is unchanged in behavior:

```
$ docuflow advanced --help

  docuflow advanced watch [stop|status|restart]    Auto-sync daemon
  docuflow advanced sync [--ai]                    One-shot sync for CI / git hooks
  docuflow advanced ui [--port N]                  Launch web UI dashboard
  docuflow advanced start                          Alias for ui
  docuflow advanced review                         Review uncommitted changes
  docuflow advanced recent                         Recent work dashboard
  docuflow advanced suggest                        First-steps guidance
  docuflow advanced update                         Self-upgrade DocuFlow

Note: every "advanced" command also works at its old top-level path
(e.g. `docuflow watch` is still valid). The "advanced" prefix is optional.
```

## CLAUDE.md auto-generation

`docuflow init` now writes a CLAUDE.md that features the **4 core MCP tools** (`query_wiki`, `ingest_source`, `wiki_search`, `read_module`) and demotes the other 11 to an "Advanced" section. The MCP server still registers all 15 tools — this is doc-only reorganisation. Existing agents using `lint_wiki`, `save_answer_as_page`, `generate_dependency_graph`, etc. continue to work unchanged.

Idempotency markers (`<!-- BEGIN DOCUFLOW -->` / `<!-- END DOCUFLOW -->`) let users re-run `docuflow init` without clobbering hand-written content above or below the DocuFlow section.

## Upgrade path

```bash
npm install -g @doquflow/cli@1.7.0
```

No commands change paths. Optionally rebuild your CLAUDE.md to pick up the new core/advanced grouping:

```bash
cd /path/to/your/project
docuflow init   # safe to re-run; updates only the DocuFlow section
```

## What's next

- **v2.0** — Package split: `@doquflow/core` (irreducible MCP server) + `@doquflow/studio` (UI/REST/daemons). `@doquflow/cli` stays as a meta-package depending on both, so existing install commands are unchanged.
- **v3.0** — Evidence-based pruning. If studio features show real use, they earned their keep. If not, they retire.

## The "no" list until v2.0

Every PR has to answer one question in one sentence: *"This makes the user's problem easier, not our system easier to build."*

- ❌ No new MCP tools (the 4-core / 11-advanced split keeps the count at 15)
- ❌ No new CLI subcommands (last additions were `query` and `ingest` in this release; both wrap existing MCP tools)
- ❌ No new UI views
- ❌ No new integrations
- ✅ Extractor precision, schema authoring UX, query-result quality, value-out docs
